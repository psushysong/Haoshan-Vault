### What the 2026 PKM tools are doing

Multiple tools (Second Brain I/O, AFFiNE, Taskade) have converged on the same pattern: an AI that runs overnight, reads what came in during the day, identifies themes, extracts key decisions, notes open loops, and surfaces connections to older material. They call it "Progressive Summarization at scale."

This is different from a search index. It is closer to a nightly editorial pass — the AI reads everything and writes a brief on what matters.

### What this vault does

The 3am journal cron runs `git diff`, writes a summary, and moves completed tasks. That is one-third of overnight processing: it captures what happened. It does not yet:

- Extract decisions made (vs. tasks completed)
- Surface connections to older material
- Note open loops that did not become tasks

The 9am morning briefing does a version of the third point ("Forgotten" section) but only from the previous day's journal entry, not across the full vault history.

### The gap

Current vault intelligence: **reactive, single-day window.**
What competitors are doing: **proactive, full-history window.**

The fix is not a vector database. It is a `connections.md` file that the morning discovery cron maintains — a running list of which notes reference similar concepts, updated incrementally as new material arrives. The COO reads this file to surface connections without scanning everything.

### Connection to April 6 note

The April 6 exploration flagged a `semantic index (even a plain index.md)` as the next move for scale. This search confirms that is the right direction — the tools winning in 2026 are not the ones with better note-taking UX, but the ones with better overnight synthesis. A maintained plaintext index is a lightweight version of what they are building.

### Possible next move

Add to the 3am journal cron:
1. After writing the git-diff summary, scan for any decisions or open loops in that day's changes.
2. Append a one-line entry to `connections.md` for any new note that connects to existing vault material.

This keeps everything in plaintext and does not require any new infrastructure.
